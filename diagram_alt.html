Alternative Network Infrastructure Diagram with Server-Based Connections

Latency Concerns:
  • Connection from public client to the server over the internet can introduce latency due to distance, network congestion, and routing.
Latency Concerns:
  • Both the public client and the internal client need to access the server over the internet, which introduces latency due to distance, network congestion, and routing.
  • Latency may also increase due to server load, network issues, and data processing delays.
Security Concerns Detailed:
  • The server is completely sandboxed to isolate it from the internal network.
  • All connections to and from the server are encrypted to prevent data interception and tampering.
  • Regular security updates and monitoring are essential to maintain the server's security.
Latency Concerns:
  • Latency may increase due to the additional hop between the server and the LED wall over the internet.
  • Data processing delays on the server and the LED wall client can contribute to latency.
Security Concerns Detailed:
  • Connection from the LED wall client to the server must be encrypted to protect data integrity and prevent unauthorized access.
  • Strict access controls and regular monitoring are required to ensure the security of the internal network.
